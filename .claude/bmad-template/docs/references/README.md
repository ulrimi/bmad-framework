# Reference Materials

This directory holds LLM-friendly reference documentation for key dependencies used in {{PROJECT_NAME}}.

## Purpose

When working with unfamiliar libraries or APIs, agents need authoritative documentation that fits in a context window. This directory stores curated, condensed reference docs that give agents the context they need without requiring web lookups.

## What Goes Here

- LLM-friendly library documentation (e.g., `fastapi-reference-llms.txt`)
- API documentation for external services
- Protocol specifications relevant to the project
- Style guides or standards documents

## How to Create Reference Docs

1. Find the official documentation for the dependency
2. Extract the most relevant sections (API reference, common patterns, gotchas)
3. Save as a plain text or markdown file: `<library>-reference-llms.txt`
4. Keep each file under 2000 lines — split large references into focused topics
